# РПМК по информатике Тверской области

Статический сайт для GitHub Pages с админкой Sveltia CMS / Decap-compatible CMS.

## Что внутри

- Главная страница.
- Раздел ВСОШ: школьный, муниципальный, региональный этапы.
- Раздел ГИА: ОГЭ, ЕГЭ, рекомендации.
- Каталог материалов.
- Новостной раздел.
- Контакты.
- Админка `/admin/` для добавления новостей, материалов и редактирования основных страниц.

## Быстрый запуск на GitHub Pages

1. Создайте новый репозиторий на GitHub, например `rpmk-informatika-tver`.
2. Загрузите в репозиторий все файлы из этой папки.
3. Откройте `admin/config.yml` и замените:

```yml
repo: OWNER/REPOSITORY
```

на свой репозиторий, например:

```yml
repo: oleg/rpmk-informatika-tver
```

4. В `admin/config.yml` замените:

```yml
site_url: "https://example.ru"
display_url: "https://example.ru"
```

на адрес вашего сайта.

5. В GitHub откройте:

```text
Settings → Pages
```

6. В блоке `Build and deployment` выберите:

```text
Source: Deploy from a branch
Branch: main
Folder: / (root)
```

7. Нажмите `Save`.

Сайт появится по адресу вида:

```text
https://OWNER.github.io/REPOSITORY/
```

## Подключение домена

1. В настройках GitHub Pages укажите домен в поле `Custom domain`.
2. Если используете поддомен, например `info.example.ru`, создайте CNAME-запись у регистратора:

```text
info.example.ru → OWNER.github.io
```

3. Если используете основной домен, например `example.ru`, создайте A-записи на IP GitHub Pages:

```text
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```

4. После проверки домена включите `Enforce HTTPS`.
5. Файл `CNAME.example` переименуйте в `CNAME` и впишите в него ваш домен без `https://`.

## Вход в админку

Админка находится по адресу:

```text
https://ваш-домен/admin/
```

Например:

```text
https://informatika-tver.ru/admin/
```

Вариант 1 — самый простой для одного администратора:

1. Откройте `/admin/`.
2. Нажмите `Sign in with Token`.
3. Создайте GitHub Personal Access Token с правом записи в репозиторий.
4. Вставьте токен в окно входа.

Вариант 2 — удобный вход через кнопку GitHub:

1. Создайте GitHub OAuth App.
2. Разверните OAuth proxy, например Sveltia CMS Authenticator на Cloudflare Workers.
3. В `admin/config.yml` раскомментируйте строку `base_url` и укажите URL worker.

## Как добавлять новости

1. Откройте `/admin/`.
2. Перейдите в раздел `Новости`.
3. Нажмите `New Новость`.
4. Заполните заголовок, описание, дату и текст.
5. Нажмите `Publish`.

После публикации CMS создаст markdown-файл в папке `_posts`, GitHub Pages пересоберёт сайт, и новость появится на главной и в разделе `/news/`.

## Как добавлять материалы

1. Откройте `/admin/`.
2. Перейдите в раздел `Материалы`.
3. Нажмите `New Материал`.
4. Заполните направление: `ВСОШ`, `ГИА`, `Методическая работа`.
5. Выберите категорию: `Школьный этап`, `Муниципальный этап`, `Региональный этап`, `ОГЭ`, `ЕГЭ`, `Рекомендации`.
6. При необходимости прикрепите файл.
7. Нажмите `Publish`.

Материал появится в каталоге `/materials/`.

## Локальный запуск

Для локальной проверки нужен Ruby.

```bash
bundle install
bundle exec jekyll serve
```

После запуска откройте:

```text
http://127.0.0.1:4000/
```

## Что заменить перед публикацией

- `OWNER/REPOSITORY` в `admin/config.yml`.
- `https://example.ru` в `admin/config.yml`.
- Домен в файле `CNAME.example`, затем переименовать файл в `CNAME`.
- Тексты в разделе `Контакты`.
- Примерные новости и материалы.

## Важное ограничение

GitHub Pages — публичный статический хостинг. Не размещайте на сайте закрытые персональные данные, непубличные протоколы, служебные документы ограниченного доступа и материалы, которые не должны быть доступны всем посетителям.
